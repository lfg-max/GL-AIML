MLS SQL Query Analysis

A breakdown of common data analysis questions for the MLS database.

1. Retrieve all rows from the album table.

SELECT * FROM album;

How It Works

SELECT * is the most basic query. It selects all columns from the specified table, and with no WHERE clause, it retrieves all rows.

2. How many rows are in the customers table?

SELECT COUNT(*) FROM customers;

How It Works

The COUNT(*) aggregate function counts the total number of rows in a table, providing a quick summary of the data volume.

3. Show only 5 sample rows from the artist table.

SELECT * FROM artist LIMIT 5;

How It Works

The LIMIT clause restricts the number of rows returned by the query. LIMIT 5 is useful for previewing a small sample of data without fetching the entire table.

4. Retrieve the unique playlists in the database.

SELECT DISTINCT playlist_name FROM playlist;

How It Works

The DISTINCT keyword eliminates duplicate rows from the result set. This query returns a clean list of each unique playlist name, even if a name appears multiple times in the table.

5. Fetch the unique artists in the database.

SELECT DISTINCT artist_name FROM artist;

How It Works

Similar to the previous query, DISTINCT ensures that each artist's name is listed only once, providing a unique roster of all artists.

6. Count the customers in the country of Brazil.

SELECT COUNT(*) FROM customers WHERE customer_country = 'Brazil';

How It Works

The WHERE clause filters rows based on a condition. Here, it restricts the count to only those rows where the customer_country column is 'Brazil'.

7. Count the number of artists.

SELECT COUNT(*) FROM artist;

How It Works

COUNT(*) is used again to get the total number of entries in the artist table.

8. Count customers in Brazil, Germany, and Canada.

SELECT COUNT(*) FROM customers WHERE customer_country IN ('Brazil', 'Germany', 'Canada');

How It Works

The IN operator allows you to specify multiple values in a WHERE clause. It is a shorthand for multiple OR conditions and makes the query cleaner.

9. Retrieve customers with IDs from 1 to 10.

SELECT * FROM customers WHERE customer_id BETWEEN 1 AND 10;

How It Works

The BETWEEN operator is inclusive and selects values within a given range. It's a more readable way of writing customer_id >= 1 AND customer_id <= 10.

10. Fetch tracks with duration between 100000-500000 and price of 0.99.

SELECT * FROM tracks WHERE milliseconds BETWEEN 100000 AND 500000 AND unit_price = 0.99;

How It Works

The AND operator combines multiple conditions in a WHERE clause. A row must satisfy *both* the duration and the price conditions to be included in the result.

11. Fetch tracks with duration between 100000-500000 or price of 0.99.

SELECT * FROM tracks WHERE milliseconds BETWEEN 100000 AND 500000 OR unit_price = 0.99;

How It Works

The OR operator also combines conditions, but a row only needs to satisfy *at least one* of the conditions to be included.

12. Get all details for customers whose first name starts with 'L'.

SELECT * FROM customers WHERE first_name LIKE 'L%';

How It Works

The LIKE operator is used for pattern matching in strings. The percent sign % is a wildcard that matches any sequence of characters. 'L%' matches any string starting with 'L'.

13. Get all details for customers whose first name ends with 'l'.

SELECT * FROM customers WHERE first_name LIKE '%l';

How It Works

By placing the wildcard % at the beginning, '%l' matches any string of characters that ends with the letter 'l'.

14. Fetch the details of the customer whose first name is Dan.

SELECT * FROM customers WHERE first_name = 'Dan';

How It Works

This is a standard equality check in a WHERE clause to find an exact match for a string value.

15. Fetch tracks whose unit price is not $0.90 and genre ID is not 18.

SELECT * FROM tracks WHERE unit_price != 0.90 AND genre_id != 18;

How It Works

The != (or <> in some SQL dialects) operator means "not equal to". This query filters for tracks that satisfy both "not equal" conditions.

16. Fetch tracks where the unit price is greater than $0.90.

SELECT * FROM tracks WHERE unit_price > 0.90;

How It Works

The > operator is a standard comparison operator used to filter for rows where a numeric value is greater than the specified value.

17. Fetch tracks whose genre id is 10.

SELECT * FROM tracks WHERE genre_id = 10;

How It Works

This is a simple equality filter on a numeric column to retrieve all tracks belonging to a specific genre ID.

18. Fetch tracks for the genre id greater than 10.

SELECT * FROM tracks WHERE genre_id > 10;

How It Works

Using the > operator again, this query fetches all tracks that belong to a genre ID numerically higher than 10.

19. Fetch invoices for Edmonton with a total greater than $8.

SELECT * FROM invoice WHERE billing_city = 'Edmonton' AND total_price > 8;

How It Works

This query combines a string equality check with a numeric greater-than check using AND to find specific, high-value invoices from a particular city.

20. Fetch invoices for Berlin or Paris on 2009-02-01.

SELECT * FROM invoice WHERE billing_city IN ('Berlin', 'Paris') AND invoice_date = '2009-02-01';

How It Works

This query effectively filters for invoices that meet two criteria: the city must be one of the two in the IN list, AND the date must match exactly.

21. Retrieve the number of invoices in each city, sorted descending.

SELECT billing_city, COUNT(*) AS NumberOfInvoices FROM invoice GROUP BY billing_city ORDER BY NumberOfInvoices DESC;

How It Works

  • GROUP BY: This clause groups rows that have the same value in billing_city into summary rows.
  • COUNT(*): This counts the number of original rows in each group.
  • ORDER BY ... DESC: This sorts the resulting groups from the highest invoice count to the lowest.

22. Get the revenue generated in each city, sorted descending by city.

SELECT billing_city, SUM(total_price) AS CityRevenue FROM invoice GROUP BY billing_city ORDER BY billing_city DESC;

How It Works

  • GROUP BY: Groups all invoices by their city.
  • SUM(total_price): For each city group, this calculates the sum of the total_price column.
  • ORDER BY billing_city DESC: Sorts the final result alphabetically by city name in reverse (Z to A).

23. Get countries with more than 5 customers.

SELECT customer_country, COUNT(*) AS NumberOfCustomers FROM customers GROUP BY customer_country HAVING COUNT(*) > 5;

How It Works

  • GROUP BY: Groups customers by their country.
  • COUNT(*): Counts the number of customers in each country group.
  • HAVING: This clause filters the *groups* created by `GROUP BY`. It keeps only those countries where the customer count is greater than 5.
