Querying Data with SQL

GL Beats Music Database

Problem Statement:

GL Beats Case Study

Business Context:

GL-Beats is a music-based startup that recently started selling music records online in multiple countries across the world. The executives at GL-Beats want to conduct a performance review and need recommendations on:

  • Which music genres should their customers be recommended?
  • Which artists need to be contracted within specific countries based on their popularity?

Objective:

As data scientists, our objective is to explore the available data by writing basic SQL queries and becoming familiar with the dataset. Additionally, we aim to answer the key business questions posed by top management, providing insights that can drive strategic decision-making.

The Heart of the Library: The `tracks` Table

This is the central table holding all information about individual songs.

  • Track Info: Name, Composer, Duration (Milliseconds), File Size (Bytes).
  • Pricing: `unit_price` determines the cost of a track.
  • Connections: Links to album, genre, media type, and playlists.

Summary & Q&A

  • The schema is well-normalized to reduce data redundancy.
  • Relationships are clearly defined using primary and foreign keys.
  • Cardinality rules enforce business logic (e.g., an invoice must have a customer).
  • A junction table (`playlisttrack`) solves the many-to-many problem for playlists.
