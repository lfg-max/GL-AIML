Phase 1: Data Ingestion & Preparation

We transform unstructured documents into a queryable, intelligent knowledge base through a sequential pipeline.

rag_system = RAG_LLM()

Initialize the core system.

rag_system.load_data()

Ingest and parse the source PDF document.

rag_system.chunk_data()

Deconstruct content into manageable segments for analysis.

rag_system.create_embeddings()

Convert text into semantic vector representations.

rag_system.setup_vector_database()

Build the high-speed retrieval index.

Phase 2: Automated Insight Generation

The system intelligently retrieves context and generates human-like answers to complex queries.

rag_system.get_answer(user_input)

The primary entry point for user queries.

create_rag_prompt()

Dynamically constructs a context-rich prompt for the AI.

get_context()

Retrieves the most relevant data chunks from the vector DB.

generate_llm_response()

Interfaces with the LLM (Gemini or Ollama) to synthesize the final answer.

Phase 3: Automated Quality Assurance

We employ an LLM-as-a-Judge model to autonomously score the generated answers for accuracy and relevance, ensuring reliable output.

rag_system.calculate_rating(question)

Initiates the full generation and evaluation cycle.

get_answer()

First, an answer is generated as in Phase 2.

rate_answer()

The generated answer is then passed to the evaluation module.

rate_groundedness()

Checks if the answer is factually supported by the source document.

rate_relevance()

Checks if the answer directly addresses the user's question.
